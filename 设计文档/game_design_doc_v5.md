# 逃生指南 · 游戏设计文档 v5.0

> 独立开发版 | 肉鸽 + 搜打撤 | 砍掉一切可做可不做的模块

---

## 一、顶层设计决策

### 1.1 单人。永远单人。

每局控制一个角色，不做队友、不做多人、不做幸存者机制。
撤离失败 = 角色死亡 = 本局结束 = 重开。
这是所有设计的出发点。

### 1.2 只有两件事：活下去，逃出去

**活下去**靠 Roguelite build——通过战斗获取遗物和特性，积累优势。
**逃出去**靠撤离决策——找到撤离点、管理时间/资源、活着离开。
两条线在同一个循环里互相拉扯：想活得好就要多打，想跑得快就要少打。

### 1.3 砍掉的模块（明确不做）

| 模块 | 原因 |
|------|------|
| 多人/队友系统 | 工作量x3，平衡难度爆炸 |
| 钥匙/道具背包 | 背包系统是另一个游戏 |
| 随机抉择事件 | 第一EA版不做，用撤离事件替代 |
| Meta永久进度 | 发EA后再做，现在做等于白做 |
| 热难度曲线（0-20级）| 数值爆炸，用简单版难度递增替代 |
| 侵蚀时间压力 | 反探索，让地图生成白做了 |
| 剧情/叙事 | 像素恐怖不需要剧本 |
| 音效/BGM | 上EA前外包或买资产 |
| 成就/统计系统 | 上EA后再加 |

## 二、核心循环

```
[局前] 选初始遗物 -> [局内] 探索房间 -> 战斗/商店 -> 选战利品 -> [撤离] 撤离 -> [局后] 解锁新内容
                    ^________________________|
```

一个 session = 一局 = 一次完整的撤离尝试
- 局前：选初始遗物（3选1）
- 局内：通过地图，经历若干场战斗，获得遗物和特性变强
- 撤离：触发撤离点后等待倒计时，成功则本局结束，失败则本局结束

## 三、局前系统：初始遗物

### 3.1 设计原则

局前选择的遗物决定本局 build 的基调，而不是强度。
不选 = 白板，游戏可以裸开，但有初始遗物体验完全不同。

### 3.2 初始遗物池（EA阶段共7个）

| 遗物名 | 效果 | 适用build |
|--------|------|---------|
| 武器大师 | 手枪全程可用，无需切换 | 全能型 |
| 疾跑者 | 移速+40%，闪避冷却-30% | 游击型 |
| 弹药狂人 | 弹药掉率+50%，弹匣+2 | 火力型 |
| 战地医生 | 每击杀1敌人回复3HP | 站桩型 |
| 暴击专精 | 暴击率+20%，暴击伤害x1.5 | 爆发型 |
| 硬汉 | 最大HP+50，受伤时硬直-50% | 坦克型 |
| 拾荒者 | 金币获取+30%，商店价格-15% | 经济型 |

### 3.3 局后解锁机制

每次成功撤离，记录本局用过的遗物。累计使用3次后，该遗物自动加入局前可选池。
初始有3个免费白板可选，随游戏次数增加逐渐解锁，最多同时选7个。

## 四、地图生成系统

### 4.1 地图规模（EA阶段）

- 地图尺寸：固定 64x48 格（1格=1米）
- 房间数量：每局 8~12 个房间（含起始房和撤离房）
- 走廊宽度：固定 2 格
- 房间尺寸：最小 8x8 格，最大 20x16 格

### 4.2 BSP算法（Binary Space Partitioning）

**分裂逻辑：**
1. 地图初始为整块矩形区域
2. 递归分裂，直到每块区域的最小边长 < 要求的最小房间尺寸
3. 分裂方向判断：若 width > height x 1.25 则竖切；否则若 height > width x 1.25 则横切；否则随机
4. 分裂位置：随机取较长边的 40%~60% 处

**叶节点 -> 房间：**
- 每个叶节点生成一个矩形房间
- 房间尺寸 = 叶节点尺寸 x (0.5~0.7)，随机取值
- 房间在叶节点内随机偏移，确保至少2格墙厚

**连接走廊（L型连接）：**
- 从父节点向下遍历，每对相邻叶节点之间生成一条L型走廊
- 走廊穿墙处生成门（1格宽）

**连通性验证：**
- 生成后用 Flood Fill 从起始房验证全房间可达
- 不可达的房强制与最近房生成额外走廊

### 4.3 房间类型分配

| 房间类型 | 数量/局 | 说明 |
|---------|---------|------|
| 起始房 | 1 | 固定在地图左下角，无敌人 |
| 撤离房 | 1 | 固定在地图右上角 |
| 战斗房 | 5~8 | 刷怪，清理后开箱 |
| 精英房 | 1~2 | 单个精英敌人，高奖励 |
| 商店房 | 1 | 消耗金币购买遗物 |

### 4.4 难度递增规则

| 已清理房间数 | 敌人HP倍率 | 敌人伤害倍率 | 每房敌人数 |
|------------|------------|------------|---------|
| 0~2 | 1.0 | 1.0 | 2~3 |
| 3~5 | 1.25 | 1.15 | 3~4 |
| 6~7 | 1.5 | 1.3 | 4~5 |
| 8+ | 1.8 | 1.5 | 5~6 |

打得越深，怪越强。

## 五、敌人系统（EA阶段共5种）

### 5.1 设计原则

EA阶段不做AI行为树。用有限状态机（FSM）+ 数值差异来区分敌人。
每种敌人有明确的核心行为和克制方式。

### 5.2 五种敌人

| 敌人 | 类型 | HP倍率 | 行为 | 克制方式 |
|------|------|--------|------|---------|
| 巡逻兵 | 普通 | x1.0 | 固定路线巡逻，看到玩家后追击 | 绕背/引开 |
| 冲锋兵 | 普通 | x0.8 | 每8秒直线冲刺，冲刺时无敌 | 冲刺前后摇，绕侧打 |
| 投掷兵 | 普通 | x0.7 | 每6秒向玩家投掷手雷（范围伤害）| 优先击杀/走位躲 |
| 精英枪手 | 精英 | x3.0 | 中距精准射击，弹匣大，2秒换弹 | 利用换弹窗口输出 |
| Boss：持盾巨人 | Boss | x8.0 | 缓慢移动，正面盾挡，间歇性震地波 | 绕后/引震地波后打侧背 |

HP倍率 = 相对基准敌人HP（100）的倍数。

### 5.3 状态机定义

```
idle（随机走动）
  -> [发现玩家：进入感知范围]
alert（原地愣1秒，视觉指向玩家）
  ->
chase（直线追击玩家）
  -> [进入攻击范围]
attack（执行攻击行为）
  -> [丢失目标：脱离感知范围5秒]
patrol（返回原巡逻路径）
```

**感知范围：** 巡逻兵5格 / 冲锋兵7格 / 投掷兵8格 / 精英枪手12格 / Boss10格

### 5.4 刷怪规则

玩家进入房间 -> 关门（门锁动画）-> 2秒后敌人从预设位置生成。
所有敌人死亡 -> 门解锁 -> 掉落箱子。

- 快速换弹：换弹速度+50%

- 精英猎手：对精英/Boss伤害+30%
- 冲锋减缓：冲锋兵冲刺间隔+50%

**史诗遗物（4个）**
- 毁灭打击：每10秒下次射击伤害x3

### 7.3 特性池（EA阶段共16个）

**普通特性（清理房间时随机1个，选1）：**
火力增强/迅捷/硬甲/弹药储备/稳枪（各+5%~+15%）

**稀有特性（精英击杀时必出1个，选1）：**
穿透射击/暴击精通/弹药狂潮/战术换弹/坚韧

**史诗特性（Boss击杀时必出2个，选1）：**
战意沸腾（HP小于30%时伤害+60%）/ 双重射击（同时开火两把枪）/ 复仇（死亡时对周围敌人造成50范围伤害，一次）

### 7.4 Build 构建思路

遗物决定长期强度曲线，特性决定即时战斗强化。
核心体验：随遗物积累，玩家逐渐从小心翼翼变成横着走——这是 Roguelite 的正反馈设计目标。

## 九、撤离系统

### 9.1 撤离流程

1. 玩家到达撤离房
2. 触发撤离事件（3种随机之一）
3. 等待90秒倒计时（期间敌人会从地图各处涌来）
4. 倒计时结束前进撤离圈 -> 撤离成功
   倒计时归零未进入 -> 撤离失败

### 9.2 撤离事件（3种随机）

| 事件 | 内容 | 选项 |
|------|------|------|
| 物资补给 | 撤离圈内出现一个免费遗物箱 | 直接领取 |
| 紧急警报 | 立即刷新3只巡逻兵 | 立即开门可跳过战斗直接撤离（损失奖励）|
| 导航员上线 | 倒计时减少15秒，获得导航特性（显示撤离房方向）| 接受/拒绝 |

### 9.3 撤离奖励

成功撤离后随机获得：随机普通遗物 / 随机稀有遗物（20%概率）/ 50金币。

## 十二、开发阶段划分

### 第一阶段：能跑通（目标：第1~6周）

| 任务 | 验收标准 |
|------|---------|
| 玩家移动+射击 | 在空场景里能打靶子 |
| BSP地图生成 | 生成房间+走廊，能走进每个房间 |
| 3种普通敌人 | 巡逻兵/冲锋兵/投掷兵，有FSM，能被打死 |
| 战利品箱子 | 敌人死后掉落箱子，打开出3选项 |
| 房间切换 | 走门进下一个房间，不穿墙 |
| 撤离系统 | 撤离房触发倒计时，能完成一次完整撤离 |

阶段终点：一个人能完成一次"选遗物->走图->战斗->撤离"的完整闭环。

### 第二阶段：让核心循环好玩（目标：第7~16周）

- 5把武器完整数值（手感/后座力/弹道调好）
- 遗物系统完整（20个遗物全部实装）
- 特性系统完整（16个特性全部实装）
- 精英敌人（精英枪手）
- Boss战（持盾巨人）
- 商店系统（购买逻辑+金币经济）
- 难度递增（按已清理房间数递增敌人参数）
- UI完善（HP条/弹药/遗物图标/HUD）
- 数值平衡（第一版通关率目标：30%~50%）

阶段终点：每局有build差异感，武器手感良好，能稳定通关。

### 第三阶段：准备EA（目标：第17~24周）

- 关卡美术（统一像素风格，地图视觉不丑）
- 敌人美术（5种敌人各一套像素动画）
- 玩家动画（移动/射击/闪避/死亡）
- 粒子特效（枪口火焰/命中血花/爆炸）
- UI美术（开始菜单/暂停/HUD/结算）
- Steam商店页（截图+描述+预告片）
- 60秒预告片（展示核心玩法+build多样性）
- Bug清理（完整测试一轮，修崩溃）

阶段终点：在Steam发布EA版，定价25元。
